### 关键概念

| 概念 | 说明 |
|------|------|
| **服务端驱动** | 对局逻辑由服务端 `lib/game.js`（继承 `Majiang.Game`）**集中运行**。客户端只负责 UI 展示和人类输入。 |
| **玩家角色** | 服务端拥有唯一的游戏状态机；客户端是"输入终端"。 |
| **通信** | 全部通过 **Socket.IO** 的 `GAME` 事件双向传输消息。 |
| **两个 Game 类** | `src/core/game.js`（客户端本地）和 `majiang-server/lib/game.js`（服务端）都继承同一基类 `Majiang.Game`，但 `call_players()` 实现完全不同。 |

### 2.2 bin/server.js — 服务器入口

**核心职责：** 创建 HTTP 服务器 + Socket.IO 服务器 + Express 路由

```javascript
// 依赖
const express  = require('express');
const session  = require('express-session');
const passport = require('../lib/passport')(auth);

// 创建 HTTP + Socket.IO
const app  = express();
const http = require('http').createServer(app);
const io   = require('socket.io')(http, {
    path: `${base}/socket.io/`,
    cors: { origin: '*' },
    pingInterval: 25000,     /* 25 秒一次心跳，避免后台标签页被浏览器限频时误判 */
    pingTimeout:  60000,     /* 60 秒无 pong 才视为断开，容忍浏览器后台限频 */
    connectTimeout: 10000
});

// 大厅
const lobby = require('../lib/lobby')(io);

// HTTP 路由
app.post(`${base}/auth/`,      passport.authenticate('local', ...));  // 本地注册
app.post(`${base}/auth/hatena`,...);  // Hatena OAuth
app.post(`${base}/auth/google`,...);  // Google OAuth
app.post(`${base}/logout`, ...);      // 登出
app.get(`${base}/status`,  ...);      // 服务器状态页
app.use(express.static(docs));        // 可选静态文件

// Socket.IO 中间件（必须登录）
io.use(wrap(session));
io.use(wrap(passport.initialize()));
io.use(wrap(passport.session()));
```

**Socket.IO 心跳参数说明：**

| 参数 | 值 | 说明 |
|------|-----|------|
| `pingInterval` | 25000ms (25s) | 服务端每隔 25 秒发送一次 ping |
| `pingTimeout` | 60000ms (60s) | 客户端必须在 60 秒内回复 pong，否则视为断线 |
| `connectTimeout` | 10000ms (10s) | 初次连接超时 |

> **设计考虑：** `pingTimeout` 设为 60 秒是为了容忍现代浏览器对后台标签页的 JavaScript 定时器限频（Chrome/Edge 对后台标签页限制为约 1 次/分钟）。若设为 8 秒，后台标签页将频繁被判定为断线。

### 2.3 lib/lobby.js — 大厅（房间管理）

**核心数据结构：**

```javascript
class Lobby {
    USER: {
        [uid]: {
            user:    { uid, name, icon },  // 用户信息
            sock:    Socket,                // WebSocket 连接（离线时为 null）
            room_no: string | undefined     // 所在房间号
        }
    }

    ROOM: {
        [room_no]: {
            uids:    [uid, uid, ...],       // 房间内用户列表（最多4人）
            game:    Game | undefined,      // 进行中的对局实例
            exptime: Date | undefined       // 房间过期时间
        }
    }
}
```

**Socket.IO 事件处理：**

| 事件 | 方向 | 处理逻辑 |
|------|------|---------|
| `connection` | Client→Server | 检查用户会话，发送 `HELLO`，初始化 USER 记录 |
| `disconnect` | Client→Server | 标记用户离线；若在房间中，通知其他成员 |
| `ROOM` | Client→Server | 三种模式：`''` 创建房间 / `room_no` 加入房间 / `room_no, uid` 踢出用户 |
| `START` | Client→Server | 房主确认开始 → 随机分座席 → 创建 `Game` 实例 → 调用 `game.kaiju()` |

**分座席逻辑（`get_socks`）：**

```javascript
get_socks(room_no) {
    let uids = this.ROOM[room_no].uids;  // 按加入顺序
    let socks = [];
    // 随机打乱 uids，填满 4 个座席（空缺的填 null，由 AI/bot 占据）
    while (socks.length < 4) {
        let uid = uids.splice(Math.random() * uids.length, 1)[0];
        socks.push(uid ? this.USER[uid].sock : null);
    }
    return socks;  // [sock_0, sock_1, sock_2, sock_3] = 东/南/西/北 ← 随机
}
```

**socks 为 null 的座席不会响应 GAME 事件，服务端会自动跳过（相当于掉线）。**

### 2.4 lib/game.js — 服务端对局（核心）

**继承关系：**
```javascript
module.exports = class Game extends Majiang.Game {
```

即服务端 Game 继承 `@kobalab/majiang-core` 中的 `Majiang.Game`，这里也是本项目 `src/core/game.js` 的基类。

**构造器：**

```javascript
constructor(socks, callback, rule, title, timer) {
    super(socks, callback, rule, title);

    this._model.player = socks.map(s => s ? s.request.user.name : '(NOP)');
    this._uid          = socks.map(s => s ? s.request.user.uid : null);
    this._players      = socks;   // 直接存 Socket，而非 Player 对象！

    this._timer = timer;  // 持牌时间：[timeout, base_time, wait_time]
}
```

**关键区别：`_players` 存的是 Socket，不是 `Majiang.Player` 对象！**

#### call_players() — 重写的核心方法

```javascript
call_players(type, msg, timeout) {
    timeout = this._speed == 0 ? 0
            : timeout == null  ? this._speed * 200 + fakelag(type)
            :                    timeout;

    this._status = type;
    this._reply  = [];
    this._seq++;  // 在客户端侧，这是个纯 UI Player，不需要 seq

    for (let l = 0; l < 4; l++) {
        let id = this._model.player_id[l];
        msg[l].seq = this._seq;
        if (this._players[id])
                this._players[id].emit('GAME', msg[l]);  // ← Socket.emit
        else    this._reply[id] = {};  // null 座席自动跳过
    }

    if (type == 'jieju') {
        this._callback(this._paipu);
        return;
    }
    this._timeout_id = setTimeout(()=>this.next(), timeout);
}
```

**与客户端 `call_players` 的核心区别：**

| 方面 | 客户端 `src/core/game.js` | 服务端 `lib/game.js` |
|------|--------------------------|---------------------|
| `_players` 内容 | `Majiang.Player` 对象（或 AI） | Socket.IO Socket |
| 调用方式 | `_players[id].action(msg[l], callback)` | `_players[id].emit('GAME', msg[l])` |
| 异步机制 | 各 Player 回调后触发 `next()` | 客户端通过 `GAME` 回复，服务端 `reply()` 收集 |
| 超时处理 | `setTimeout` 后强制 `next()` | 同上，另有 timer 计时器 |
| 消息序号 | 无 | `_seq` 防重入 |

#### reply() — 收集客户端回复

```javascript
reply(id, reply) {
    if (reply.seq != this._seq) return;  // 消息序号不匹配 → 丢弃
    if (this._reply[id]) return;         // 已回复 → 忽略
    // 收集回复，4人齐备后调度 next()
    this._reply[id] = reply;
    if (this._reply.filter(x=>x).length < 4) return;
    if (! this._timeout_id)
            this._timeout_id = setTimeout(()=>this.next(), 0);
}
```

#### 计时器系统

服务端比客户端多了**持牌时间限制**：

```javascript
// timer 数组含义：
// timer[0] = 每手牌最大思考时间 (秒)
// timer[1] = 对局初始剩余时间 (秒)
// timer[2] = 每局重启后的等待时间 (秒)

// 在 call_players 中，根据 type 决定是否设限：
function get_timer(type, limit, allowed, wait) {
    if (type == 'jieju')                       return;        // 终局无限制
    if (type.match(/^(kaiju|hule|pingju)$/))   return wait ? [wait] : null;  // 等待时间
    else                                        return [limit, allowed];      // 倒计时
}
```

#### 其他重写方法

- `notify_players(type, msg)`: 同 call_players，但不等待回复（用于牌局状态同步、语音播放）
- `say(name, l)`: 向所有客户端发送语音播放指令
- `connect(sock)`: 断线重连后恢复对局（发送完整牌局历史）
- `disconnect(sock)`: 标记玩家离席。记录断线时间（`_disconnectTime`），全部离席时不会立即终止对局，而是给予 **30 秒重连宽限期**；超过宽限期才调用 `stop(callback)` 终止
- `delay(callback, timeout)`: 带错误捕获的延迟执行

**断线重连机制：**

```
客户端断线
  → Socket.IO 检测心跳超时（pingTimeout: 60s）
  → 触发 'disconnect' 事件
  → lobby.disconnect(sock)
    → 删除 USER[uid].sock（保留 USER 记录和 room_no）
    → game.disconnect(sock)
      → _players[id] = null
      → 记录 _disconnectTime[id] = Date.now()
      → 若所有玩家断线：检查是否在 30s 宽限期内 → 不终止
      → 自动回复当前 seq（防止对局卡住）
  ↓
客户端自动重连（reconnection: true）
  → Socket.IO 发起重连（reconnectionDelay: 500ms）
  → 触发 'connection' 事件
  → lobby.connect(sock)
    → 检测 USER[uid] 存在且 sock 为空 → 重连路径
    → USER[uid].sock = sock
    → game.connect(sock)
      → _players[id] = sock（恢复连接）
      → 清除 _disconnectTime[id]
      → 发送 'START' + kaiju（含完整 log）恢复对局状态
```

### 2.5 lib/passport.js — 认证模块

支持三种登录方式：
- **本地认证**：用户名 + 密码（密码固定为 `*`，主要靠 session）
- **Hatena OAuth**：需要 `hatena.json` 配置文件
- **Google OAuth**：需要 `google.json` 配置文件

用户对象结构：`{ uid, name, icon }`，其中 `uid` 在无 OAuth 时回退为 `sessionID`。

### 2.6 bin/client.js — Bot 客户端

命令行机器人，可自动加入房间进行对局：

```bash
majiang-bot -r A1234 -n '麻雀ロボ' http://127.0.0.1:4615/server
```

- 使用 `socket.io-client` 连接服务器
- 内部使用 `@kobalab/majiang-ai` 的 AI Player 做决策
- 流程：POST 登录 → WebSocket 连接 → `ROOM` 加入 → 等待 `GAME` 消息 → 调用 `player.action(msg, callback)`

## 三、客户端架构

### 3.1 src/js/netplay.js — 联机页面逻辑

**文件入口：** `webpack.config.js` → `netplay: './src/js/netplay.js'` → 产出 `dist/js/netplay-2.5.1.js`

**页面：** `dist/netplay.html`（由 `src/html/page/netplay.pug` 编译）

**核心流程：**

```
页面加载
  → 500ms 后调用 init()
  → 创建 Socket.IO 连接
    sock = io('/', {
        path: `${base}/server/socket.io/`,
        transports: ['websocket', 'polling'],
        reconnection: true,
        reconnectionDelay: 500,
        reconnectionDelayMax: 5000,   // 重连间隔上限，防止重连风暴
        reconnectionAttempts: Infinity
    });

  → 注册事件监听
    sock.on('HELLO', hello);           // 用户信息
    sock.on('ROOM',  room);            // 房间状态
    sock.on('START', start);           // 对局开始
    sock.on('END',   end);             // 对局结束
    sock.on('ERROR', file.error);
    sock.on('disconnect', ...);        // 断线处理
    sock.on('reconnect', ...);         // 重连成功日志
    sock.on('reconnect_error', ...);   // 重连失败日志

  → 收到 HELLO
    hello(user)
      ├── user == null → 显示登录表单
      └── user 有效 → 显示房间表单（対局中则不切换 UI）
```

### 3.2 HELLO 事件处理

```javascript
function hello(user) {
    if (!user) {
        // 未登录 → 显示登录界面
        $('body').attr('class','title');
        show($('#title .login'));
        return;
    }
    // 已登录 → 显示房间表单
    myuid = user.uid;
    /* 重连时如果已在対局中（body class=board），不要切回 file 界面，
     * 避免破坏対局 UI。服务端会随后发送 START 重新初始化 Board。 */
    if ($('body').attr('class') !== 'board') {
        show($('#file .netplay form'));
        fadeIn($('body').attr('class','file'));
    }
    // 设置头像、用户名
    $('#file .netplay img').attr('src', user.icon).attr('title', user.uid);
    $('#file .netplay .name').text(user.name);
}
```

**disconnect 事件处理（重连时）：**

```javascript
sock.on('disconnect', (reason) => {
    /* 対局中不断线提示，等待自动重连 */
    if ($('body').attr('class') !== 'board') {
        hide($('#file .netplay form.room'));
    }
});
sock.on('reconnect', (attemptNumber) => {
    console.log('Socket.IO 重连成功 (尝试次数: ' + attemptNumber + ')');
});
sock.on('reconnect_error', (error) => {
    console.log('Socket.IO 重连失败:', error.message);
});
```

> **重连流程：** 客户端断线后 Socket.IO 自动尝试重连（`reconnectionDelay: 500ms`，上限 `5000ms`，无限次尝试）。重连成功后，服务端依次发送 `HELLO` → `START` → `GAME(kaiju+log)` 恢复对局状态。`hello()` 中检测 `body.class === 'board'` 避免切换 UI。

### 3.3 ROOM 事件处理 — 房间管理

```javascript
function room(msg) {
    // msg = { room_no: 'A1234', user: [{uid, name, icon, offline}, ...] }

    $('body').attr('class','room');
    $('#room input[name="room_no"]').val(msg.room_no);

    // 显示房间内用户列表
    for (let user of msg.user) {
        // 我是房主 或 这条是我的 → 显示"退出"按钮
        if (msg.user[0].uid == myuid || user.uid == myuid)
            show(quit_button);
        // 离线标记
        if (user.offline) r.addClass('offline');
    }

    // 只有房主显示规则选择器和"开始对局"按钮
    if (msg.user[0].uid == myuid) {
        show(rule_select, timer_input, start_button);
    }
}
```

### 3.4 START 事件处理 — 对局开始

```javascript
function start() {
    // 创建 UI Player（人类操作者）
    const player = new Majiang.UI.Player($('#board'), pai, audio);

    // 创建 Board View
    player.view = new Majiang.UI.Board($('#board .board'), pai, audio,
                                         player.model, {});

    // 创建 GameCtl（UI 控制面板）
    const gameCtl = new Majiang.UI.GameCtl($('#board'), 'Majiang.pref',
                                              null, player, player._view);
    gameCtl._view.no_player_name = false;

    let players = [];  // 其他玩家的用户信息

    // ⚠️ 关键：移除旧的 GAME 监听，重新注册
    sock.removeAllListeners('GAME');
    sock.on('GAME', (msg) => {
        if (msg.players) {
            // 玩家信息更新
            players = msg.players;
        }
        else if (msg.say) {
            // 语音播放指令
            player._view.say(msg.say.name, msg.say.l);
        }
        else if (msg.seq) {
            // seq > 0 的消息需要回复（要我出牌/操作）
            if (seq && msg.seq != seq) location.reload();  // 消息错序 → 刷新
            player.action(msg, (reply = {}) => {
                reply.seq = msg.seq;
                sock.emit('GAME', reply);  // ← 回复给服务端
                seq = msg.seq + 1;
            });
        }
        else {
            // seq 为 0 的消息只需要执行，无需回复（如 qipai 配牌）
            player.action(msg);
            // 如果是 kaiju 消息且含历史 log，重放历史操作
            if (msg.kaiju && msg.kaiju.log) {
                let log = msg.kaiju.log.pop();
                for (let data of log) {
                    player.action(data);
                }
            }
        }
        // 更新所有玩家名称显示
        player._view.players(players);
    });
}
```

## 四、通信协议

### 4.1 Socket.IO 事件一览

```
客户端 ────────────────────────→ 服务端
│                                  │
│  connection ───────────────────→ │  建立连接
│  ←──────────── HELLO ────────── │  返回用户信息（或 null）
│                                  │
│  ROOM('') ────────────────────→ │  创建房间
│  ←──────────── ROOM ──────────── │  返回房间号 + 用户列表
│                                  │
│  ROOM('A1234') ───────────────→ │  加入房间
│  ←──────────── ROOM ──────────── │  返回更新后的房间信息
│                                  │
│  START('A1234', rule, timer) ──→ │  开始对局（仅房主）
│  ←──────────── GAME ──────────── │  kaiju 消息（带完整 log）
│  ── GAME(reply) ──────────────→ │  客户端回复
│  ←──────────── GAME ──────────── │  下一轮消息
│  ...                             │  ...
│  ←──────────── END ───────────── │  对局结束
│                                  │
│  ROOM('A1234', uid) ──────────→ │  踢出用户（仅房主）
│  ←──────────── ROOM ──────────── │  更新房间信息
│                                  │
│  disconnect ──────────────────→ │  断线（心跳超时 60s）
│  reconnect ───────────────────→ │  自动重连（客户端）
│    ├── HELLO ──────────────────→│  恢复用户信息
│    ├── START ──────────────────→│  重新初始化 Board
│    └── GAME(kaiju+log) ────────→│  重放对局状态
│  ←──────────── ERROR ─────────── │  错误消息
```

**断线重连时间线：**

| 阶段 | 时间 | 说明 |
|------|------|------|
| 服务端发 ping | 每 25s | `pingInterval: 25000` |
| 客户端回复 pong | ≤60s | 超时则视为断线 |
| 客户端尝试重连 | 500ms 后 | `reconnectionDelay: 500` |
| 重连间隔上限 | 5s | `reconnectionDelayMax: 5000` |
| 服务端宽限期 | 30s | 所有玩家断线后 30s 内不终止对局 |

### 4.2 GAME 消息格式

服务端通过 `notify_players` 和 `call_players` 向每个座席发送**独立的消息**（`msg[l]`，l=0/1/2/3 对应东/南/西/北）。

#### 消息结构

```javascript
// 所有 GAME 消息的通用结构
{
    seq: Number,      // 消息序号（>0 表示需要回复，0 表示单向通知）
    timer: [         // 可选：计时器设置
        limit,        //   本回合最大时间（秒）
        allowed       //   累计剩余时间（秒）
    ],
    // 以下是互斥的 type 字段（只有一个存在）：
    kaiju:    { ... },  // 开局
    qipai:    { ... },  // 配牌
    zimo:     { ... },  // 摸牌
    dapai:    { ... },  // 舍牌（+等待其他家操作）
    fulou:    { ... },  // 副露
    gang:     { ... },  // 杠
    gangzimo: { ... },  // 杠后摸岭上牌
    kaigang:  { ... },  // 开杠宝牌指示牌
    hule:     { ... },  // 和了
    pingju:   { ... },  // 流局
    jieju:    { ... },  // 半庄结束
}
```

#### 各消息类型详情

**kaiju（开局）：**
```javascript
{
    kaiju: {
        id:     Number,  // 我方在 player_id 中的序号
        rule:   Object,  // 规则配置
        title:  String,  // 牌局标题
        player: [name0, name1, name2, name3],  // 四家名称
        qijia:  Number,  // 起家位置 (0-3)
        log:    [[...]], // 历史操作日志（断线重连时用到）
    }
}
```

**qipai（配牌）：**
```javascript
{
    qipai: {
        shoupai: String[],  // [手牌串, ...] 空位用 '' 占位
        baopai:  String,    // 宝牌指示牌串
        fubaopai:String,    // 里宝牌指示牌串（或 ''）
        defen:   Number[],  // [points, ...] 各持分
    }
}
```

**zimo（摸牌）：**
```javascript
{
    zimo: {
        l:        Number,  // 当前座位 (0-3)
        p:        String,  // 摸到的牌 ('' 表示非当前玩家)
        shoupai:  String[], // 手牌串
        baojia:   Number,  // 流动的听牌信息
    }
}
```

**dapai（舍牌）：**
```javascript
{
    dapai: {
        l:        Number,  // 出牌座位
        p:        String,  // 舍出的牌
        d:        String,  // 打出后的手牌（用于副露者判断）
        shoupai:  String[], // 各手牌串
        he:       String[], // 各牌河串
    }
}
```

### 4.3 客户端回复格式

```javascript
// 回复给服务端的对象，必须带 seq：
{
    seq: Number,     // 必须等于收到的 seq

    // 以下根据当前状态互斥：
    hule:  Boolean,  // zimo 时自摸宣告 / dapai 时荣和宣告
    gang:  String,   // 杠的牌（暗杠）
    dapai: String,   // 舍出的牌（'m1_' 表示摸切）

    fulou: {
        type: String,     // 'chi' | 'peng' | 'daminggang'
        mianzi: String,   // 面子串
        p:      String,   // 副露后舍出的牌
        d:      String,   // 副露后手牌
    },

    // pingju 时
    shoupai: String[], // 手牌（用于流局判断）
    fenpei:  Number[], // 点差分配

    // jieju 时
    // 无需回复，只需空 {} 带 seq
}
```

### 4.4 players 消息（玩家信息同步）

```javascript
// 由 notify_players 发送，msg.say = undefined, msg.seq = undefined
{
    players: [
        { uid, name, icon },  // player 0
        { uid, name, icon },  // player 1
        null,                 // player 2 (空缺)
        { uid, name, icon },  // player 3
    ]
}
```

### 4.5 say 消息（语音播放）

```javascript
// 由服务端 game.say(name, l) 发送
{
    say: {
        l:    Number,  // 发言席位
        name: String,  // 语音名称（如 'ron', 'tsumo', 'pon' 等）
    }
}
```

## 五、本地对局 vs 联机对局 差异对比

### 5.1 架构对比

| 维度 | 本地模式 | 联机模式 |
|------|---------|---------|
| 入口文件 | `src/js/index.js` | `src/js/netplay.js` |
| 页面 | `dist/index.html` | `dist/netplay.html` |
| Game 实例位置 | 客户端浏览器 | Node.js 服务端 |
| Player 类型 | `Majiang.UI.Player` + `Majiang.AI` | `Majiang.UI.Player`(仅自己) |
| 通信方式 | 进程内函数调用 | Socket.IO WebSocket |
| 规则来源 | localStorage + rule.json | 房主选择，随 START 消息发送给服务端 |
| 牌谱存储 | `localStorage['Majiang.game']` | `localStorage['Majiang.netplay']` |

### 5.2 call_players 对比

```javascript
// 本地模式（src/core/game.js）：
call_players(type, msg, timeout) {
    for (let l = 0; l < 4; l++) {
        let id = this._model.player_id[l];
        this._players[id].action(msg[l], reply => this.reply(id, reply));
    }
}

// 联机模式（majiang-server/lib/game.js）：
call_players(type, msg, timeout) {
    for (let l = 0; l < 4; l++) {
        let id = this._model.player_id[l];
        this._players[id].emit('GAME', msg[l]);  // 发送到浏览器！
    }
}
```

### 5.3 Player 构造函数参数

```javascript
// 本地模式
new Majiang.Game(
    [ uiPlayer, ai1, ai2, ai3 ],  // 4 个 Player 对象
    callback,
    rule
);

// 联机模式（服务端）
new Game(
    [ sock_0, sock_1, sock_2, sock_3 ],  // 4 个 Socket（可能为 null）
    callback,
    rule,
    title,
    timer
);
```

## 六、技能集成（已实现）

### 6.1 实现状态

超能力职业技能支持已在服务端完整实现。`majiang-server/lib/game.js` 在构造器中初始化 `SkillManager`，并在所有关键状态变换处调用技能钩子。

### 6.2 涉及的文件

| 文件 | 改动 |
|------|------|
| **majiang-server/lib/game.js** | ⭐ 核心：构造器中初始化 SkillManager，`call_players`/`notify_players`/状态转换处触发技能钩子；新增 `CHARACTER` 事件处理、角色选择流程 |
| **majiang-server/lib/lobby.js** | START 事件中解析规则中的技能相关字段（`技能模式`、`角色分配方式` 等），传给 Game 构造器 |
| **src/js/netplay.js** | 客户端处理角色选择流程（`CHARACTER` 事件 → 弹窗 → 确认 → 回复服务端） |
| **src/skill/\*** | 复用到服务端，无需修改 |

### 6.3 联机角色选择流程

```
  服务端 kaiju() → SkillManager 初始化
    → dealCharacters(DRAW_4)  // 每人抽4张
    → notify_players('CHARACTER', options)

  客户端收到 CHARACTER
    → CharacterSelector 弹窗
    → 玩家选择 + AI 自动选
    → emit('CHARACTER', choice)

  服务端收集所有选择
    → confirmCharacter()
    → 继续 qipai()
```

### 6.4 对局中技能触发

```
  服务端 _skill_trigger(timing, context)
    → 无玩家交互的技能：直接执行
    → 有玩家交互的技能：在 GAME 消息中插入 skill_prompt 字段
      → 客户端展示 SkillPrompt UI
      → 玩家选择后回复
    → 服务端 respondToSkill() → 继续流程
```
